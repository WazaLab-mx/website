🔧 Manual Console Script

If the bookmarklet doesn't work, copy and paste this script into your browser console (F12 → Console tab):

// Upload Fix Script - Paste this into browser console (F12) console.log('🛡️ Upload Fix: Removing browser restrictions...'); // Remove file input restrictions document.addEventListener('DOMContentLoaded', function() { setTimeout(() => { const fileInputs = document.querySelectorAll('input[type="file"]'); fileInputs.forEach(input => { input.removeAttribute('accept'); input.removeAttribute('webkitdirectory'); input.setAttribute('multiple', 'true'); console.log('📁 File input fixed:', input); }); }, 1000); }); // Override FormData restrictions if (window.FormData) { const originalFormData = window.FormData; window.FormData = function(form) { const formData = new originalFormData(form); const originalAppend = formData.append; formData.append = function(name, value, filename) { try { return originalAppend.call(this, name, value, filename); } catch (e) { console.warn('FormData restriction bypassed:', e); return originalAppend.call(this, name, value || '', filename); } }; return formData; }; } // Override fetch restrictions if (window.fetch) { const originalFetch = window.fetch; window.fetch = function(url, options = {}) { if (options.method === 'POST' || options.method === 'PUT') { options.mode = 'cors'; options.credentials = 'same-origin'; if (!options.headers) { options.headers = {}; } delete options.headers['Content-Security-Policy']; delete options.headers['X-Content-Type-Options']; } return originalFetch.call(this, url, options); }; } // Fix form restrictions document.querySelectorAll('form').forEach(form => { form.removeAttribute('onsubmit'); form.setAttribute('enctype', 'multipart/form-data'); }); console.log('✅ Upload restrictions removed! Option 2: Full Python Script (All platforms)

# Remove all policy restrictions python3 remove-policies.py --type=all # Remove only browser policies python3 remove-policies.py --type=browser # Dry run (see what would be changed) python3 remove-policies.py --dry-run

🔍 Troubleshooting

If uploads still don't work:

  • Completely close and restart your browser
  • Clear browser cache and cookies
  • Try a different browser (Chrome, Firefox, Safari, Edge)
  • Check if antivirus software is blocking uploads
  • Disable browser extensions temporarily
  • Try incognito/private browsing mode

Common solutions by error type:

  • CORS errors: The bookmarklet fixes these automatically
  • File type restrictions: The script removes accept attributes
  • Security policy blocks: Use the system-level scripts
  • Network blocks: Check firewall and proxy settings

📞 Still Need Help?

If none of these solutions work:

  • Check the browser console (F12) for specific error messages
  • Try uploading from a different device or network
  • Contact your system administrator if on a corporate network
  • Check if the knowledge base service itself is experiencing issues

Debug Information

When reporting issues, include:

  • Browser type and version
  • Operating system
  • Error messages from browser console
  • File types you're trying to upload
  • Network environment (corporate, home, public)
